## Abstract

This paper describes the design and implementation of **Emergent**, a Unity simulation in which wolf packs and rabbit populations interact across procedurally generated terrain. The simulation uses a custom **QuadGrammar** system — a spatial grammar operating over a quadtree index — to generate ecologically structured environments. Agent behavior is governed by individual data profiles (behavioral, physical, social) rather than scripted sequences, so that the observed dynamics arise from rule interactions rather than authored outcomes. The paper documents the architectural decisions, the emergent phenomena observed, and the implications for procedural world generation in ecological simulation.

---

## 1. Motivation

Most simulation systems face a tradeoff: scripted behavior is predictable but brittle; random behavior is varied but incoherent. The question this project attempts to answer is whether a middle path exists — behavior that is rule-governed enough to be believable, and varied enough to produce surprise.

The ecological context (predator-prey dynamics) was chosen because it has well-understood analytical models (Lotka-Volterra equations) against which emergent behavior can be benchmarked. If the simulation produces population curves that qualitatively match known ecological theory *without explicitly encoding those curves*, it demonstrates genuine emergence rather than parameterized imitation.

## 2. The QuadGrammar System

### 2.1 Motivation for Grammar-Based Generation

Standard procedural terrain generation (Perlin noise, midpoint displacement) produces visually organic surfaces but lacks ecological structure. A forest generated by noise has no concept of "this area is sheltered," "this ridge is a territory boundary," or "water collects here." These structural properties matter for agent behavior — wolves need territory markers, rabbits need cover.

A grammar-based approach allows rules to encode ecological meaning:

```
OPEN_PLAIN → [DENSE_BRUSH (cover=high) + CLEARING (visibility=high)] if: near_water
RIDGE      → TERRITORY_BOUNDARY if: elevation > threshold
```

Rules operate on spatial regions, not pixels. The result is terrain with **structural semantics** that agents can query.

### 2.2 Quadtree as Spatial Index

The quadtree structure serves two purposes:

1. **Multi-scale generation**: Grammar rules fire at different quadtree depths, producing broad structural features at coarse resolution and fine detail at deep resolution. The same rule system generates both "this is wolf territory" (coarse) and "these specific bushes provide cover" (fine).

2. **Efficient spatial queries**: Agent perception (WolfSenses, RabbitAI) requires frequent "what is near me" queries. A quadtree reduces these from O(n) to O(log n), enabling larger agent populations at real-time speeds.

### 2.3 Grammar Rule Structure

Rules have the form:
```
REGION_TYPE → [CHILD_REGIONS] | ATTRIBUTE_UPDATES
  conditions: [spatial, ecological, stochastic]
  weight: float
```

The parser applies rules iteratively until a termination condition (max depth, no applicable rules) and then passes the final quad tree to the rendering and simulation systems.

## 3. Wolf Agent Architecture

### 3.1 The Profile System

Rather than scripting wolf behavior directly, each wolf is instantiated from a set of data profiles:

- **WolfBehaviorProfile** — aggression range, fear threshold, curiosity, territory attachment
- **WolfColorProfile** — coat markings, pattern distribution (determines visual identity)
- **WolfBodyProportions** — limb ratios, skull shape (determines mesh)
- **WolfAffinity** — relationship weights to other wolves (updated at runtime)

Visual identity and behavioral identity are generated from the same random seed, so wolves that look similar tend to behave similarly. This produces natural-feeling social groups without explicit group-scripting.

### 3.2 Pack Dynamics

**WolfPack** maintains a roster of wolves and computes emergent pack-level properties from individual states:

- Territory is the union of individual wolves' active range weighted by territory attachment score
- Hunting coordination emerges from individual wolves responding to prey-found signals from packmates, not from a centralized "hunt" command
- Leadership is not assigned — it emerges as the wolf most likely to initiate movement is followed most frequently

### 3.3 Procedural Animation

Wolves have no keyframe animations. **WolfProceduralAnimator** performs real-time bone manipulation based on:

- Current velocity and acceleration
- Terrain normal (so paws contact uneven ground correctly)
- Behavioral state (stalking gait vs. sprint vs. rest)

The result is locomotion that naturally adapts to terrain geometry, including QuadGrammar-generated elevation variation.

## 4. Rabbit Agent Architecture

Rabbits are simpler agents by design — they are the variable the wolves act on. Their behavior is:

1. **Idle/forage** when no predator is detected
2. **Flee** when WolfSenses detection radius is entered
3. **Shelter** when near cover (queries quadtree for cover=high regions)

The asymmetry is intentional: complex predators, simple prey. This is ecologically realistic and produces more interesting emergent dynamics than balanced complexity on both sides.

## 5. Observed Emergent Phenomena

Running the simulation with varying initial parameters has produced several dynamics not explicitly programmed:

**Territory stabilization**: Wolf packs reliably develop stable territory boundaries even without explicit territory-marking rules. The boundary emerges from where pack members' active ranges intersect — they naturally avoid traveling far beyond where prey density supports their energy cost.

**Boom-bust cycles**: Rabbit population spikes are followed by wolf population growth, which depletes rabbit populations, which contracts wolf pack sizes. This qualitatively matches Lotka-Volterra predictions. The simulation did not encode these equations.

**Pack fragmentation**: When rabbit population crashes, large packs split — individual wolves with high independence scores disperse rather than compete for scarce resources within the pack. This was not scripted; it emerges from the behavioral profile system.

**Grammar-behavior coupling**: Rabbits concentrate in QuadGrammar-generated brush regions (cover=high). Wolf hunting patterns adapt to this, circling brush perimeters rather than entering directly. Neither species was given explicit instructions about cover — they each responded to the terrain semantics independently.

## 6. Implications for Procedural World Design

The key insight from this project: **grammar-generated worlds produce qualitatively different emergent behavior than noise-generated worlds** because grammar encodes structure that agents can reason about.

A rabbit in a noise-terrain has no reason to prefer one area over another except surface coloring. A rabbit in a QuadGrammar-terrain can detect that it is in a cover=high region and adjust behavior accordingly. This single addition produces realistic sheltering and den behavior without any scripting.

For game designers and simulation researchers, this suggests a principle: **the semantics of your world generation system become the vocabulary of your agent behavior**. More structurally meaningful world generation enables more meaningful agent behavior at no additional behavioral scripting cost.

## 7. Future Directions

- Extending QuadGrammar to include temporal rules (seasonal variation, resource depletion)
- Adding a third species tier (apex predator, or competing herbivore) to test stability of emergent pack dynamics
- Exporting simulation telemetry to study whether emergent population curves can be used to validate or challenge classical ecological models
