These papers document research-in-practice: built alongside working software, tested in real classrooms, grounded in a perceptual experience (grapheme-color synesthesia) that most interface designers don't have access to.

## Papers

### [Synesthetic Perception as a Design Methodology for AI Interfaces](./synesthetic-ai-interface-design.md)
How does involuntary color-number mapping (grapheme-color synesthesia) inform the design of AI-driven interfaces? This paper argues that synesthetic perception is a form of parallel multimodal processing — and that designing AI systems for synesthetic users reveals latent design failures invisible to neurotypical evaluation.

*Related project: [rainbowbrain-dalle](https://github.com/sfdimarco/rainbowbrain-dalle)*

### [Emergent Behavior and Procedural Ecology in Simulation](./emergent-behavior-procedural-ecology.md)
A study of emergent AI behavior in a Unity simulation environment using a custom QuadGrammar terrain system and wolf/rabbit agent architectures. Explores how simple local rules produce complex systemic behavior — and what that means for educational game design.

*Related project: [SpatialToleranceCompression](https://github.com/sfdimarco/SpatialToleranceCompression)*

### [Visual Coding Environments for K-8 Learners: Design Principles](./visual-coding-k8-design-principles.md)
Ten years of teaching K-8 students to code, distilled into design principles for visual programming environments. Covers the gap between Blockly-style tools and real code, and how to build the bridge without losing students on either side.

*Related project: [p5-blockly-coding](https://github.com/sfdimarco/p5-blocky-coding)*

## Background

I have **grapheme-color synesthesia** — digits involuntarily appear as specific colors. This is not a metaphor. It is a neurological feature that shapes every interface decision I make, every AI prompt I write, and every educational tool I build.

The SYN color system (0=cyan, 1=red, 2=blue, 3=yellow, 4=green, 5=orange, 6=purple, 7=pink) runs as a constant across all my projects. These papers explain why.
